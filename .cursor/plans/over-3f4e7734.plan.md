# Overview Page Updates

## Changes Overview

This plan will update the Overview page ([app/page.tsx](app/page.tsx)) to add a Favorites section and improve content card previews to show MDX content for words.

## Implementation Steps

### 1. Add Favorites List to Constants or Data Layer

Add a list of favorite content slugs to [lib/constants.ts](lib/constants.ts) or [lib/data.ts](lib/data.ts). This list will contain the slugs of items you want to feature in the Favorites section.

Example:

```typescript
export const FAVORITE_SLUGS = [
  'words/diplomacy',
  'posts/architecture-of-silence',
  'library/thinking-fast-and-slow',
];
```

### 2. Export Favorites Array

In [lib/data.ts](lib/data.ts), add a direct export that filters favorite content items at build time:

```typescript
export const favorites = ALL_CONTENT.filter(item => 
  FAVORITE_SLUGS.includes(item.slug)
);
```

This follows the same pattern as `ALL_CONTENT` - computed once at module load.

### 3. Update Overview Page Structure

Modify [app/page.tsx](app/page.tsx) to:

- Import the favorites data
- Add a new "Favorites" section after the intro, before the "Recently Tended" section
- Update both sections to render MDX content for word-type items

### 4. Implement MDX Content Rendering for Words

For word-type content in both sections:

- Create the MDX component from `word.content` using the same pattern from [app/words/page.tsx](app/words/page.tsx):
  ```typescript
  const getMDXContent = new Function(word.content);
  const MDXContent = getMDXContent({ React, ...runtime }).default;
  ```

- Render `<MDXContent />` in the preview area with `line-clamp-3`
- Keep description/summary for posts, library, and engineering items

### 5. Add Section Header for Favorites

Create a section header similar to "Recently Tended" but labeled "Favorites" with appropriate styling to match the existing design.

## Files to Modify

1. **[lib/constants.ts](lib/constants.ts)** - Add FAVORITE_SLUGS array
2. **[lib/data.ts](lib/data.ts)** - Add getFavorites helper function
3. **[app/page.tsx](app/page.tsx)** - Add Favorites section and update card rendering to support MDX for words

## Key Design Decisions

- Word-type content will show MDX preview (actual content)
- Other types (posts, library, engineering) will continue showing description/summary
- Favorites section will appear before "Recently Tended" section
- All other aspects (labels, dates, styling) remain unchanged
