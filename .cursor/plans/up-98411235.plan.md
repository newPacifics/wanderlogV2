# Update Engineering Detail Page

## Changes Required

### 1. Update Velite Schema

**File:** [`velite.config.ts`](velite.config.ts)

Make the demo-specific fields optional in the engineering collection schema:

- Change `componentKey: s.string()` to `componentKey: s.string().optional()`
- Change `description: s.string()` to `description: s.string().optional()`
- Change `instructions: s.string()` to `instructions: s.string().optional()`
- Add `content: s.mdx()` to enable MDX content rendering

### 2. Update Engineering Detail Page

**File:** [`app/engineering/[id]/page.tsx`](app/engineering/[id]/page.tsx)

Transform the page to:

- Import MDX runtime dependencies (like the posts detail page does)
- Create MDX component from `demo.content` 
- Conditionally render the description section only if `description` exists
- Conditionally render the demo component only if `componentKey` exists
- Conditionally render the instructions section only if `instructions` exists
- Add the MDX content rendering section with proper prose styling (similar to posts)

The layout will be:

1. Header (title, date, type badge)
2. Description (if exists)
3. Demo component (if componentKey exists)
4. Instructions (if exists)
5. **MDX content** (always rendered)
6. Tags footer

This approach allows engineering posts to be:

- Interactive demos with components + MDX content
- Plain MDX posts (like regular posts) without demo components
